## 2NF
* `CourseSchedule` 中將一門課的多個上課時段記錄為多筆資料（用流水號或唯一 ID 區分）

* `CourseSelection` 與 `Student`、`Course` 都有主鍵關聯，避免資料重複與異常

## 3NF
| 表格                                        | 2 NF → 3 NF 變動重點                                                          | 為何需要調整？                                                        |
| ----------------------------------------- | ------------------------------------------------------------------------- | -------------------------------------------------------------- |
| `Course`                                  | **無變動**                                                                   | 每一非鍵屬性僅依賴 PK `course_no`，不存在非鍵 → 非鍵的依賴。                        |
| `Student`                                 | 引入 `dept_id`（FK）並建立 `Department` 表                                        | 系所名稱會重複；若未拆分，`student_dept` 變更須回填所有紀錄 → 更新異常。                  |
| `CourseSchedule`                          | 去除 `course_building`，新增 `Room`／`Building` 表；改用 surrogate PK `schedule_id` | `course_building` 其實由 `course_room` 決定（O313 屬於 綜教館 等）→ 典型轉移依賴  |
| `CourseTeacher`                           | 引入 `teacher_id`，建立 `Teacher` 主檔                                           | 避免同一教師姓名重複更新，並預留教師其他屬性空間。                                      |
| 其餘 (`CurriculumField`, `CourseSelection`) | 僅把人名改為 `student_id`、`teacher_id` 等 FK；無轉移依賴問題                             | —                                                              |
